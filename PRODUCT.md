# Product

## Register

product

## Users

Multi-device users who switch between desktop and mobile browsers. Developers and curious explorers who want agentic AI that can code, use tools, and manipulate files. They're in a casual context — trying things, exploring what the AI can do, iterating quickly. Not enterprise workflows or mission-critical dashboards.

## Product Purpose

OpenKot is a web-based UI for OpenCode — agentic AI that doesn't just chat but acts. Chat interface with superpowers: code editing, terminal access, file exploration, tool execution. Success looks like ChatGPT's simplicity with Replit's "just try it" energy. Users should feel invited to experiment, not intimidated by complexity.

## Brand Personality

Playful, approachable, experimental. Think Replit's friendliness meets Claude's clarity. Three words: **inviting, spunky, unpretentious**. The interface should feel like a playground where things just work, not a serious developer tool that demands respect.

## Anti-references

- Generic AI wrapper sites with glassmorphism and gradients
- "AI-generated" aesthetic clutter (hero metrics, gradient text, side-stripe borders)
- Corporate SaaS darkness (navy blue, cards everywhere, lifeless)
- Hacker/cyberpunk neon-on-black terminal aesthetic

## Design Principles

1. **Chat-first, not tool-first** — The conversation is the hero. Tools, editors, and terminals support the chat, never overshadow it.
2. **Mobile parity** — Desktop and mobile get equal design attention. No "mobile version" feels like a concession.
3. **Invite experimentation** — UI should hint "try this" rather than "configure that". Discoverability over settings.
4. **Playful but not childish** — Animations, micro-interactions, and delight exist, but the tool remains competent and trustworthy.
5. **Simplicity scales** — Complex features (terminal, file explorer, MCP config) stay simple until the user needs them. Progressive disclosure, not feature walls.

## Accessibility & Inclusion

Standard defaults. No specific WCAG requirements beyond typical browser accessibility. Reduced motion support preferred but not mandated.
